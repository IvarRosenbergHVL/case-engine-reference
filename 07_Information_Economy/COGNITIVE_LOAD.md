# Cognitive Load

Cognitive Load describes how much mental effort players must spend to process, remember, compare, and interpret case information.

## Purpose

A case should challenge players without overwhelming them.

Cognitive Load helps designers distinguish satisfying complexity from exhausting confusion.

## Definition

Cognitive Load is the amount of attention, memory, comparison, and interpretation effort required to use information in the archive.

## Load sources

Cognitive Load MAY increase through:

- many actors
- similar names or roles
- dense timelines
- many locations
- overlapping documents
- technical terminology
- unresolved contradictions
- hidden dependencies
- high noise
- visually complex documents
- long reading passages

## Load types

| Type | Description |
|---|---|
| Memory load | Information players must remember. |
| Comparison load | Information players must compare across documents. |
| Interpretation load | Meaning players must infer. |
| Context load | Background knowledge required. |
| Visual load | Effort required to read or inspect visual material. |
| Coordination load | Effort required for a group to share findings. |

## Good load

Good Cognitive Load creates meaningful investigation.

It asks players to connect, compare, and discuss information.

## Bad load

Bad Cognitive Load creates confusion without payoff.

It makes players lose track of basic facts, miss critical clues unfairly, or spend effort on material that does not support discovery.

## Normative requirements

Cognitive Load SHOULD match target difficulty and play time.

Critical reasoning SHOULD NOT depend on excessive unmanaged load.

Context clues SHOULD reduce unnecessary context load.

Document structure SHOULD help players manage dense information.

## Validation questions

- How much must players remember at once?
- How many documents must be compared to reach key insights?
- Is difficulty caused by reasoning or by disorganization?
- Are technical terms explained or contextualized?
- Does the group have enough structure to share findings?
