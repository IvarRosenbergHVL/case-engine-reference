# Context Cost

Context Cost describes the background understanding players need in order to interpret a clue correctly.

## Purpose

Some clues require domain knowledge, procedural knowledge, local knowledge, technical knowledge, or social context.

If this context is not available inside the archive, the case may become unfair.

## Definition

Context Cost is the amount of background understanding required to interpret an information unit, evidence exposure, or discovery node.

## Context sources

Context may come from:

- context clues
- document headers
- explanatory notes
- procedural forms
- media articles
- manuals
- timelines
- maps
- witness explanations
- institutional documents

## Cost levels

| Level | Description |
|---|---|
| None | Common understanding is enough. |
| Low | Minor context helps but is not required. |
| Moderate | Players need provided context to interpret the clue well. |
| High | Interpretation depends on specific embedded context. |
| Excessive | Required context is missing, too obscure, or too hard to connect. |

## Context cost vs clue visibility

A clue may be visible but have high Context Cost.

Players may see it without knowing why it matters.

A case can manage this by placing context clues elsewhere in the archive.

## Normative requirements

Required context SHOULD be embedded in player-facing materials.

A critical clue SHOULD NOT depend on high Context Cost without supporting context clues.

Context clues SHOULD appear early enough to support fair interpretation.

Excessive Context Cost SHOULD trigger validation warnings or repair.

## Validation questions

- What background knowledge is required?
- Is that knowledge included in the archive?
- Can players connect the context to the clue?
- Does missing context make the solution unfair?
- Is the context natural in-world?
