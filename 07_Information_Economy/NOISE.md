# Noise

Noise is information that supports realism, texture, ambiguity, or natural document form without directly advancing the solution.

## Purpose

Real archives contain irrelevant, partial, repetitive, and routine information.

Noise helps documents feel authentic and prevents every sentence from feeling like a puzzle clue.

However, unmanaged noise can make a case unfair or exhausting.

## Definition

Noise is player-facing information that is not required for the intended solution but affects reading, attention, realism, or interpretation.

## Noise types

| Type | Description |
|---|---|
| Procedural noise | Routine official detail. |
| Atmospheric noise | Setting, tone, and world texture. |
| Personal noise | Everyday human detail. |
| Administrative noise | Filing numbers, addresses, schedules, signatures. |
| Technical noise | Lab, system, or process detail that is realistic but not central. |
| Social noise | Rumor, gossip, emotion, or casual communication. |
| Benign ambiguity | Details that may seem relevant but are not decisive. |

## Useful noise

Useful noise:

- makes documents feel real
- slows premature certainty
- creates natural reading patterns
- supports genre atmosphere
- gives important clues a realistic environment

## Harmful noise

Harmful noise:

- hides critical facts unfairly
- creates unsupported theories
- consumes too much play time
- makes documents feel padded
- increases cognitive load without payoff
- competes with intended discovery

## Normative requirements

Noise SHOULD have a purpose.

Noise SHOULD match document type and source realism.

Noise SHOULD NOT be used as the only mechanism for hiding critical clues.

A case SHOULD control total noise according to target difficulty and play time.

## Validation questions

- What purpose does this noise serve?
- Does it support realism or merely add volume?
- Does it make critical clues unfairly hard to find?
- Does it create unintended hypotheses?
- Is total package noise appropriate for play time?
