# Roadmap

The Case Engine Reference will be developed through milestones.

## M0 — Specification Framework

Goal: establish the structure and governance of CER.

Deliverables:

- repository structure
- master outline
- style guide
- writing guidelines
- specification language
- terminology
- versioning
- ADR process
- RFC process
- architecture overview

## M1 — Foundations

Goal: define the core theory of case construction.

Topics:

- vision
- design philosophy
- information-first model
- investigation theory
- case as information system
- player-facing vs hidden information

## M2 — Case Architecture

Goal: define the core graph models.

Topics:

- Truth Graph
- Timeline Graph
- Relationship Graph
- Evidence Graph
- Perception Graph
- Discovery Graph
- Case Graph
- Document Distribution Matrix

## M3 — Evidence System

Goal: define evidence, clues, claims, contradictions, red herrings, and evidence strength.

## M4 — Document System

Goal: define document types, document roles, document templates, and document realism.

## M5 — Player Experience

Goal: define player psychology, hypothesis progression, group reasoning, and difficulty.

## M6 — AI Engine

Goal: define implementation-independent agent architecture and LangGraph mapping notes.

## M7 — Validation

Goal: define conformance tests, quality gates, solvability checks, and repair strategy.

## M8 — Rendering and Facilitator Mode

Goal: define printable outputs, DOCX/PDF packaging, visual design, solution packs, hints, and hosted play.

## M9 — Complete v1.0 Release

Goal: publish a complete v1.0 reference suitable for implementation.
