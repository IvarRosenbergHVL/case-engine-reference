# Document Distribution

Document Distribution defines how evidence, context, contradictions, noise, and red herrings are spread across a case package.

## Purpose

A case package should feel like an archive, not a sequence of puzzle cards.

Document Distribution ensures that information is spread in a way that supports discovery, fairness, difficulty, and realism.

## Definition

Document Distribution is the allocation of planned evidence exposures and document roles across a set of player-facing and facilitator-only artifacts.

## Distribution goals

A distribution plan SHOULD ensure:

- all critical facts are exposed
- no single document solves the case
- clue density matches difficulty
- document roles are balanced
- red herrings are fair
- context clues are available before they are required
- suspect profiles remain balanced through early and mid discovery
- final confirmation feels earned

## Distribution dimensions

| Dimension | Description |
|---|---|
| Evidence coverage | Which facts and fragments are exposed. |
| Role balance | Mix of anchor, clue, context, red herring, eliminator, and atmosphere. |
| Source diversity | Variety of people, institutions, systems, and formats. |
| Strength distribution | How strong evidence appears across the package. |
| Discovery timing | When players are likely to notice or understand the information. |
| Noise distribution | How non-critical realism is spread. |
| Spoiler containment | Whether decisive information is delayed, hidden, or contextualized. |

## Normative requirements

A complex case SHOULD include a document distribution plan.

Critical evidence SHOULD be distributed across independent sources or formats.

Context clues SHOULD appear early enough to support later interpretation.

Red herrings SHOULD be distributed so they create plausible hypotheses without overpowering the solution.

## Validation questions

- Are all critical facts represented?
- Are evidence exposures concentrated too heavily in one document?
- Are document roles balanced?
- Does the distribution support the intended difficulty?
- Does the package contain enough realism without burying core clues?
