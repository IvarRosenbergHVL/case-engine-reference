# Document Realism

Document Realism defines whether an artifact feels like it belongs in the case world.

## Purpose

Realistic documents help players suspend disbelief and treat the archive as something assembled rather than authored.

Realism also limits what information a document can plausibly contain.

## Definition

Document Realism is the degree to which a document's content, style, source, metadata, format, and limitations match its in-world origin.

## Realism dimensions

| Dimension | Question |
|---|---|
| Source realism | Would this source create this document? |
| Content realism | Would this document contain this information? |
| Style realism | Does the tone match the source and period? |
| Format realism | Does layout match document type? |
| Limitation realism | Does the document omit what it should not know? |
| Archive realism | Why is this document in the case file? |

## Realism constraints

A police report should not contain private thoughts unless recorded as testimony.

A witness statement should not contain objective truth the witness cannot know.

A newspaper article should not behave like a puzzle hint unless intentionally stylized.

A technical report should be precise but may still require interpretation.

## Normative requirements

A document SHOULD respect the knowledge limits of its source.

A document SHOULD use a style appropriate to its source, time, and purpose.

A document SHOULD have a plausible reason to appear in the archive.

A document SHOULD NOT include information solely because the player needs it, unless the source plausibly has that information.

## Validation questions

- Would this source know this?
- Would this source write this way?
- Would this document exist in the archive?
- Does the document reveal too much for its type?
- Does realism make the clue harder without making it unfair?
