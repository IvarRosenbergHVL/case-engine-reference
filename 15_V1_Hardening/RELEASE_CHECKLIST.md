# Release Checklist

This file is the release-facing checklist for CER 1.0.

The detailed release candidate checklist is maintained in:

- `15_V1_Hardening/RC_CHECKLIST.md`

## Required before CER 1.0

- V1 scope is frozen.
- Core sections are present.
- Minimal schemas are present.
- Compliance documents are present.
- Critical failure modes are present.
- Audit checklist exists.
- Terminology decisions are documented.
- Normative language rules are documented.
- Known blocking issues are resolved or tracked.

## Release rule

CER 1.0 SHOULD only be published after the V1 audit confirms that the repository is coherent, scoped, and implementable.
