# Tests Index

Tests define how a case engine implementation can validate generated cases against CER.

Test IDs use the format:

```text
TEST-0001
```

## Initial test families

- truth completeness
- timeline consistency
- evidence redundancy
- cross-document reasoning
- suspect balance
- discovery progression
- document realism
- facilitator safety
- export completeness

Tests SHOULD reference one or more rules.
