# Case Style Guide

A Case Style Guide is the shared production reference for the visual, textual, and material identity of a case package.

## Purpose

The Case Style Guide ensures that all artifacts feel like they belong to the same investigation archive.

It gives human producers, AI tools, renderers, designers, and reviewers a common reference.

## Definition

A Case Style Guide is a case-specific design and writing guide that defines how artifacts should look, read, and behave across the package.

It is not the case truth.

It is a production control document.

## Recommended sections

A Case Style Guide SHOULD define:

| Section | Purpose |
|---|---|
| Case identity | Name, visual tone, and overall archive feel. |
| Institutional identity | Departments, agencies, logos, stamps, headers. |
| Typography | Font roles, hierarchy, spacing, and constraints. |
| Color palette | Official colors, paper tones, highlight colors, warning colors. |
| Document conventions | Headers, metadata placement, signatures, file numbers. |
| Writing style | Formality, terminology, language level, voice by source type. |
| Photo style | Camera realism, lighting, framing, sharpness, evidence visibility. |
| Map and diagram style | Symbols, labels, scale, orientation, annotation rules. |
| Digital artifact style | Screenshots, exports, metadata, device or platform conventions. |
| Aging and material effects | Paper texture, wear, stains, folds, scan noise, compression. |
| Accessibility | Legibility, contrast, print usability. |
| Forbidden style patterns | Effects, colors, or motifs that should not be used. |

## Case-specific but not truth-specific

The Case Style Guide may be specific to a generated case package.

However, it SHOULD NOT introduce new case facts, evidence, suspects, methods, motives, or solution information.

Those belong in the case model and evidence system.

## Use in production

Artifact Production Specifications SHOULD reference the Case Style Guide when visual or textual continuity matters.

Prompt derivation MAY use the Case Style Guide to create tool-specific instructions.

Human reviewers SHOULD use the Case Style Guide when approving artifacts.

## Normative requirements

A case with many visual or AI-generated artifacts SHOULD define a Case Style Guide.

The Case Style Guide SHOULD be available before bulk artifact production.

Repeated visual elements SHOULD be controlled by the guide.

The guide SHOULD distinguish requirements from recommendations.

## Validation questions

- Does the artifact follow the case style guide?
- Does the guide support both human and AI production?
- Are logos, stamps, templates, and visual conventions stable?
- Does the guide avoid adding case truth?
- Can reviewers use it to approve or reject artifacts?
