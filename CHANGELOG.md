# Changelog

All notable changes to the Case Engine Reference will be documented in this file.

The format loosely follows Keep a Changelog, adapted for specification work.

## [1.0.0-rc.0] — CER V1 Release Candidate — In progress

### Added

- Foundations for document-based investigation case design.
- Case Architecture models including truth, timeline, relationship, evidence, perception, discovery, and constraints.
- Evidence System covering evidence fragments, reliability, redundancy, critical facts, contradictions, red herrings, and means/motive/opportunity.
- Document System covering document models, metadata, roles, realism, specifications, planning, distribution, validation, rendering, and packaging.
- Representative Document Types for police, forensic, and digital materials.
- Discovery System covering hypotheses, inference chains, elimination, suspicion, confidence, and discovery curves.
- Information Economy covering information density, clue density, noise, visibility, cognitive load, discovery cost, and context cost.
- Critical V1 Failure Modes.
- Validation framework covering validation reports, rule engine, severity, waivers, repair recommendations, certification, and release decision.
- Rendering framework covering templates, assets, visual clue preservation, print readiness, export formats, audience separation, package manifests, rendering validation, and accessibility.
- Artifact Production framework covering production specifications, human control, workflow, quality gates, recommendations, prompt derivation, visual continuity, style guides, capability profiles, and producer selection.
- Case Engine reference architecture from planning to export.
- Production Graphs covering dependency tracking, change impact analysis, and partial regeneration.
- Minimal V1 JSON schemas.
- Compliance model with levels, self assessment, and implementation guide.
- V1 Hardening section with scope freeze, audit checklist, terminology decisions, normative language guidance, and release checklist.

### Changed

- Updated README to reflect V1 hardening status and current repository structure.
- Updated Roadmap to align with V1 scope freeze.
- Updated Master Outline to match the actual V1 section structure.

### Notes

CER V1 is now in hardening. Further work should focus on audit, consistency, cross-references, and release readiness rather than new major concepts.
