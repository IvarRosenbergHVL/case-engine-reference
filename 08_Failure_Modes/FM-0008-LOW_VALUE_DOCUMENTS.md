# FM-0008 — Low Value Documents

## Description

Low Value Documents occurs when a case contains too many documents that do not add meaningful evidence, context, realism, contradiction, corroboration, or discovery value.

It is a document system failure rather than a pure volume issue.

## Player impact

Players learn that many documents do not matter.

This weakens trust in the archive and makes important documents feel arbitrary rather than earned.

## Detection signals

- documents cannot be assigned a clear role
- documents repeat the same information without a new function
- documents exist only to make the package feel larger
- many documents have no evidence exposure or context purpose
- removing a document has no effect on discovery, realism, or pacing
- players stop reading carefully because documents appear low value

## Common causes

- generation pipeline produces documents before planning their roles
- package size is prioritized over information design
- realism is simulated through quantity rather than function
- document type library is used mechanically
- no document distribution matrix review

## Repair strategies

- remove documents without a role
- merge weak documents into stronger composite artifacts
- assign real evidence, context, or corroboration purpose
- reduce repeated procedural filler
- strengthen metadata and document source logic
- re-run document distribution validation

## Severity

Medium.

It becomes high severity when low-value documents hide critical clues or break player trust.
