# Elimination

Elimination defines how players can reject or weaken suspects, hypotheses, timelines, methods, or interpretations.

## Purpose

A homicide investigation game is often solved by narrowing possibilities.

Elimination makes wrong theories useful: they create discussion, then become weaker as players compare them against evidence.

## Definition

Elimination is the evidence-supported reduction, rejection, or downgrading of a possible explanation.

Elimination does not always prove a hypothesis impossible. It may only make it less likely than alternatives.

## Elimination targets

Elimination MAY apply to:

- suspect
- motive
- method
- opportunity
- timeline
- witness claim
- object interpretation
- relationship interpretation
- scene interpretation

## Elimination types

| Type | Description |
|---|---|
| Direct contradiction | Evidence conflicts with a claim or theory. |
| Opportunity failure | Time, location, or access makes a theory unlikely. |
| Means failure | Actor lacks required tool, access, ability, or knowledge. |
| Motive collapse | Proposed motive is weakened or reframed. |
| Better explanation | Another hypothesis explains more evidence. |
| Source unreliability | A supporting claim loses credibility. |
| Context correction | New context changes the meaning of earlier evidence. |

## Normative requirements

Major red herrings SHOULD have elimination paths.

A suspect SHOULD NOT be eliminated only by facilitator explanation.

Elimination SHOULD be supported by player-facing evidence.

Elimination SHOULD distinguish impossible from merely unlikely.

## Validation questions

- What wrong hypotheses are players expected to reject?
- Which evidence supports each rejection?
- Is the elimination fair?
- Does it arrive too early or too late?
- Does it depend on outside knowledge?
