# Contradiction Taxonomy

Contradictions are conflicts between facts, claims, records, interpretations, or expectations.

## Purpose

Contradictions create investigative friction.

They make players compare sources instead of accepting every document at face value.

## Definition

A contradiction is a conflict between two or more information units that cannot all be fully true in the same interpretation.

## Contradiction types

| Type | Description |
|---|---|
| Fact-claim contradiction | A claim conflicts with objective truth. |
| Claim-claim contradiction | Two sources make incompatible claims. |
| Time contradiction | Timing, order, or duration conflicts. |
| Location contradiction | Location or movement conflicts. |
| Object contradiction | Ownership, custody, state, or presence conflicts. |
| Identity contradiction | Role, relationship, or identity conflicts. |
| Interpretation contradiction | Same evidence supports different explanations. |
| Procedural contradiction | An official record conflicts with expected procedure. |

## Intentional vs accidental contradictions

Intentional contradictions are designed.

Accidental contradictions are defects.

A validator SHOULD help distinguish them.

## Fair contradictions

A fair contradiction:

- is discoverable
- has an explanation
- affects reasoning
- can be evaluated from the archive
- does not require author-only knowledge

## Unfair contradictions

An unfair contradiction:

- is caused by generation inconsistency
- cannot be resolved or interpreted
- misleads without purpose
- creates impossible timelines
- breaks the intended solution

## Normative requirements

Important contradictions SHOULD be represented in structured form.

Intentional contradictions SHOULD have a resolution path.

Accidental contradictions SHOULD be repair targets.

## Validation questions

- Is this contradiction intentional?
- What source is wrong, partial, or misleading?
- Can players evaluate the conflict?
- Does the contradiction support discovery or merely confuse?
