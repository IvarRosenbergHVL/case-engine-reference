# Context Clues

Context Clues provide the domain knowledge needed to interpret other evidence fairly.

## Purpose

Some cases depend on specialized knowledge: professions, chemicals, legal wording, medical terms, local geography, institutional routines, technology, or social customs.

If players need that knowledge, the archive must provide it.

## Definition

A Context Clue is an evidence fragment whose main purpose is to make another clue interpretable.

It may not directly identify the solution, but it gives players the background required to reason fairly.

## Common sources

Context clues MAY appear in:

- newspaper articles
- workplace safety sheets
- manuals
- brochures
- expert notes
- course material
- trade magazines
- local history texts
- public notices
- marginalia
- technical appendices
- police background summaries

## Good context clues

A good context clue:

- feels natural in the archive
- explains only what players need
- does not read like a puzzle hint
- can be connected to a suspect, location, profession, or object
- supports interpretation without solving the case alone

## Bad context clues

A bad context clue:

- appears as an obvious exposition dump
- tells players exactly what to conclude
- is disconnected from the case world
- replaces evidence reasoning
- arrives only after the solution is impossible without it

## Normative requirements

Required domain knowledge SHOULD be embedded as context clues.

A critical fact SHOULD NOT depend on outside research.

Context clues SHOULD be distributed naturally across plausible document types.

## Validation questions

- Does the case require specialized knowledge?
- Is that knowledge included in player-facing materials?
- Is the context clue natural in-world?
- Does it support reasoning without over-explaining?
