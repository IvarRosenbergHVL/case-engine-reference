# Red Herrings

Red Herrings are misleading but fair evidence structures that support plausible wrong theories.

## Purpose

Red herrings create discussion, uncertainty, and hypothesis testing.

They should make the investigation richer without making the solution arbitrary.

## Definition

A Red Herring is an intentionally misleading information structure that points toward a wrong suspect, method, motive, timeline, or interpretation while remaining explainable after the solution is known.

## Red herring categories

| Category | Description |
|---|---|
| Suspect red herring | Makes an innocent actor look suspicious. |
| Motive red herring | Suggests a motive that is not central. |
| Opportunity red herring | Suggests access or timing that later weakens. |
| Method red herring | Suggests the wrong mechanism. |
| Relationship red herring | Overemphasizes a relationship. |
| Timeline red herring | Supports an incorrect sequence. |
| Context red herring | Uses true context to suggest wrong meaning. |

## Fair red herrings

A fair red herring:

- is supported by real evidence
- has an in-world explanation
- can be eliminated or downgraded
- does not require ignoring stronger evidence
- does not exist only to trick the player

## Unfair red herrings

An unfair red herring:

- is random
- has no resolution
- depends on withheld information
- is stronger than the intended solution evidence
- creates frustration rather than investigation

## Normative requirements

Red herrings SHOULD be explainable after the solution.

A red herring SHOULD have a discovery role.

A red herring SHOULD NOT be more strongly supported than the intended solution unless later evidence clearly reframes it.

## Validation questions

- What wrong theory does this red herring support?
- How can players later downgrade it?
- Is it fair or arbitrary?
- Does it balance suspicion without stealing the solution?
